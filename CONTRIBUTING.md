# 贡献指南

## 如何贡献

我们欢迎各种形式的贡献!在贡献之前,请仔细阅读以下内容。

## 代码贡献

### 1. Fork 项目

首先,将项目fork到您的GitHub账号。

### 2. 创建分支

为您的修改创建一个新分支:

```bash
git checkout -b feature/your-feature-name
```

### 3. 进行修改

- 确保代码风格一致
- 添加必要的注释
- 在代码文件头部添加版权声明

### 4. 版权声明格式

所有代码文件必须包含以下版权头:

**PHP 文件格式:**
```php
<?php
/**
 * TXCWeb - [文件描述]
 *
 * Original Copyright (c) 2024-2026 田小橙 (TXC)
 * Modifications: Copyright (c) [年份] [您的名字] <您的邮箱>
 * Licensed under the MIT License
 *
 * [您的修改说明]
 */
```

**CSS 文件格式:**
```css
/**
 * TXCWeb - [文件描述]
 *
 * Original Copyright (c) 2024-2026 田小橙 (TXC)
 * Modifications: Copyright (c) [年份] [您的名字] <您的邮箱>
 * Licensed under the MIT License
 *
 * [您的修改说明]
 */
```

**JavaScript 文件格式:**
```javascript
/**
 * TXCWeb - [文件描述]
 *
 * Original Copyright (c) 2024-2026 田小橙 (TXC)
 * Modifications: Copyright (c) [年份] [您的名字] <您的邮箱>
 * Licensed under the MIT License
 *
 * [您的修改说明]
 */
```

### 5. 提交 Pull Request

- 提供清晰的PR描述
- 说明您修改的内容和原因
- 确保所有测试通过

## 贡献者署名规则

### 自动署名

当您的Pull Request被合并后,您将被自动添加到项目的贡献者列表中。

### 手动添加署名

如果您希望手动添加贡献者信息,请在 `README.md` 文件中更新"贡献者"部分:

```markdown
## 贡献者

- **田小橙 (TXC)** - 项目创始人及主要开发者
- **您的名字** - [贡献描述,例如:添加了新功能X,修复了问题Y]
```

### 代码修改署名示例

如果您修改了现有代码文件,请遵循以下格式:

**文件头部添加:**
```php
/**
 * Original: Copyright (c) 2024-2026 田小橙 (TXC)
 * Modified by: [您的名字] on [日期]
 * 
 * 修改内容:
 * - 优化了代码性能
 * - 添加了新的功能
 * - 修复了[问题描述]
 */
```

**代码注释中标注修改:**
```php
// Modified by [您的名字] - [日期]: 添加了新的函数
function newFunction() {
    // 您的代码
}
```

## 文档贡献

如果您希望改进文档:

1. Fork 项目
2. 修改 `README.md` 或创建新的文档文件
3. 提交 Pull Request
4. 在PR描述中说明您的文档改进

文档贡献同样会被记录在贡献者列表中。

## 非代码贡献

我们也欢迎以下形式的贡献:

- 报告 Bug
- 提出新功能建议
- 改进UI/UX设计
- 翻译文档
- 撰写教程或示例代码

## 行为准则

请遵守以下行为准则:

1. **尊重他人** - 尊重所有贡献者和社区成员
2. **建设性反馈** - 提供建设性的反馈和建议
3. **友善沟通** - 保持友善和专业的沟通方式
4. **遵守协议** - 尊重MIT License及其要求

## 代码审查

所有提交的代码都需要经过审查。审查者会关注:

- 代码质量和风格
- 版权声明是否完整
- 是否符合MIT License要求
- 是否有潜在的安全问题
- 性能影响

## 获取帮助

如果您在贡献过程中遇到问题:

1. 查看 [README.md](README.md) 获取项目信息
2. 查看 [LICENSE_SUMMARY.md](LICENSE_SUMMARY.md) 了解许可协议
3. 创建 Issue 提问
4. 联系项目作者

## 许可协议

所有贡献都必须遵循MIT License。通过提交贡献,您同意:

1. 您的贡献将使用MIT License
2. 您有权提交此贡献
3. 您的贡献不会侵犯他人权利
4. 您的代码包含完整的版权声明
